This PR addresses all 14 findings in issue #87, ensuring robust validation, optimized database aggregation, and synchronized documentation.

## Changes Included
- **MediaPicker**: Fix kind inference logic for snake_case media assets to prevent video misclassification.
- **usePlatformPageSeo**: Always emit WebSite node to prevent dangling `isPartOf` references.
- **Database**: Add `CHECK (type IN ('faq', 'how_to'))` constraint to `platform_content_components`.
- **Validation**: Enforce strict `&&` presence checks for structured content (FAQ and How-To steps) in platform docs and blog editors.
- **Types**: Extract shared `PlatformBlogPostRequestBody` and `PlatformDocRequestBody` types into `server/types/platform-content.ts` to reduce duplication.
- **Auth**: Replace redundant `hasAdminRole` function with standard `hasBetterAuthAdminRole` import in dev login route.
- **Analytics**: Refactor `returningVisitors` and `uniqueVisitors` accumulation logic to use a subquery approach `COUNT(DISTINCT visitor_id)`, accurately deduplicating visitors across multi-day ranges and avoiding SQLite parameter limits (~999 vars).
- **Platform Content**: Fix `ensureMediaAssetExists` arguments to explicitly define the field path for `how_to_steps` images.
- **MCP**: Require `body` and `components` properties in `BLOG_RECORD_SCHEMA` and `DOC_RECORD_SCHEMA`.
- **Middleware**: Introduce robust length truncation for referrer (max 2048) and user-agent (max 1024) strings prior to pageview logging.
- **Docs**: Synchronize `AGENTS.md` and `CLAUDE.md` guidelines.

Fixes #87
